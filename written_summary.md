Written Summary

The Habitat CLI advances the local simulation in one-second ticks. Each tick fetches current solar irradiance from Kepler, calculates energy generated by online solar modules, subtracts module power demand from batteries, advances the tick counter, and saves local state.

Solar generation uses:

`generatedKwh = powerGenerationKw * (wPerM2 / 900) * 0.5 / 3600`

Generated energy is added to local batteries without exceeding `energyStorageKwh`. If Kepler's solar endpoint fails, the tick stops before creating or changing simulation state.

Workflow:

```bash
bun run src/index.ts solar status
bun run src/index.ts register --name "Artemis Ridge"
bun run src/index.ts construct small-solar-array
bun run src/index.ts tick 180
bun run src/index.ts tick 1
bun run src/index.ts module status
```

The first tick sequence completes construction. The following tick shows `generatedKwh` and the updated stored battery energy.
